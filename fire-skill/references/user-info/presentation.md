# 用户信息展示规则

所有产品先展示可比的基础信息；再展示产品特有信息。手机号统一脱敏，身份证号、完整实名信息和 Token 均不展示。

## 基础信息

| 产品 | 用户 ID | 风火号 | 用户名/昵称 | 手机号 | 创建时间 | 账号状态 |
|------|---------|--------|-------------|--------|----------|----------|

| 显示项 | 规则 |
|--------|------|
| 产品 | 显示“通用版”或“一站式发货” |
| 手机号 | 显示为 `138****8000`；若接口已脱敏则直接使用返回值 |
| 创建时间 | Unix 秒级时间戳转换为 `YYYY-MM-DD HH:mm:ss` |
| 账号状态 | 接口提供时转为可读文本；未提供则显示 `-` |

## 通用版补充信息

按需展示订购情况、账户开通摘要、短信余额、推荐人、客户端信息和业务员信息。除非用户明确说明业务需要，不展示身份证号、实名信息或完整订单详情。

## 一站式发货补充信息

展示最近登录时间、主/子账号身份、组织 ID 和组织关系：

- 主账号：列出下属子账号数量；用户要求明细时再展示成员表。
- 子账号：展示所属主体组织名称。

`loginTime` 为毫秒级时间戳，应先换算后再格式化。

## 无结果与多产品结果

- 单产品无结果：`当前产品下未找到该用户。`
- 两个产品均查询时：分别展示“通用版”和“一站式发货”结果或无结果提示，不合并字段。
