# 多产品用户信息查询

查询任务统一称为“产品用户信息查询”，产品决定用户体系和接口。当前支持通用版与一站式发货；所有产品复用当前聊天项目中 `fire-skill/.token` 的 Token 生命周期。

## 路由

| 用户意图 | 产品 | 接口文档 |
|----------|------|----------|
| 通用版用户、通用版客户、风火递通用版 | 通用版 | [generic/base.md](generic/base.md) |
| 一站式发货用户、OSD 用户、一站式用户 | 一站式发货 | [osd/base.md](osd/base.md) |
| 仅说“查用户”或“查用户基本信息” | 未指定 | 先询问用户选择通用版或一站式发货 |

用户明确要求“两个产品都查”时，按两个产品分别查询并分区输出；不要合并为同一用户记录，也不要因一个产品查无结果而推断另一个产品无结果。

## 查询流程

1. 按 [共享 API 规则](../shared-api.md) 检查 Token。
2. 识别产品；产品不明确时，先询问“查询通用版还是一站式发货用户？”。
3. 从用户输入提取查询值和类型；必要时询问用户 ID、手机号或风火号。
4. 阅读对应产品的 `base.md`，按其请求方式调用接口。
5. 用户不存在时，明确说明当前产品下无匹配用户；不要跨产品自动推断。
6. 按 [用户信息展示规则](presentation.md) 输出。

## 后续扩展

后续在对应产品目录内增加资料类型即可，无需增加新的主功能：

```text
user-info/
├── generic/
│   ├── base.md
│   ├── settings.md      # 未来：配置
│   └── addresses.md     # 未来：收件地址
└── osd/
    ├── base.md
    ├── settings.md      # 未来：配置
    └── addresses.md     # 未来：收件地址
```
